2.1.1 - Keyboard Official 2.1.1 WCAG Docs

The intent of this success criterion is to ensure that, wherever possible, content can be operated through a keyboard or keyboard interface (so an alternate keyboard can be used).

Who does this benefit?

  • People who are blind
  • People with low vision
  • Some people with hand tremors find using a mouse very difficult and therefore usually use a keyboard
  • People who must use alternate keyboards or input devices that act as keyboard emulators

How to meet this criterion

  • Ensure keyboard control for all functionality

Note:
The keyboard navigation in question only refers to the use of native keyboard navigation, ex. 'Tab' & 'Shift+Tab'
