4.1.2 - Name, Role, Value Official 4.1.2 WCAG Docs

This success criterion aims to help people using assisstive technology understand all components in context

Who does this benefit?

  • People using assistive technology, such as screen readers, screen magnifiers, and speech recognition software

How to meet this criterion

  • Using aria-label to provide an invisible label where a visible label cannot be used
  • Using aria-labelledby to provide a name for user interface controls
More details on how to meet this criterion

Examples

A close button in a pop-up box


Code example:

<button aria-label="Close" class="btn-close"></button>

Labelling a table of content


Code example:

<td aria-labelledby="header-item">Good Printing</td>

<td aria-labelledby="header-documents">1,000</td>
